Explanation of Installation and Building Process

Installation:

I didn't have to download any files since I used the CDN for Materialize CSS and JavaScript. I linked the Materialize CSS in the <head> section and the Materialize JavaScript files at the end of the <body>.

Building the Page:

To build the page, I utilized Materialize’s grid system to structure the layout. I used cards and buttons, which are part of the Materialize component library. For example, I added cards with content to explain how I installed the framework and the difficulties I encountered.

Challenges and Solutions

Responsiveness:

Initially, the layout wasn’t displaying correctly on smaller screens. I fixed this by ensuring I used the grid system properly and the col classes.

Images Not Responsive:

The images I used weren’t resizing automatically. I fixed this by adding the responsive-img class to the <img> tags, which made the images resize according to the screen size.

Centering Content:

The content wasn’t centered properly on mobile devices. To solve this, I added Materialize’s center-align class to the elements that needed to be centered.

Hosting on GitHub Pages:

The page was hosted on GitHub Pages, which allowed me to quickly deploy the site without needing a separate web hosting provider. It also provided me with HTTPS support for secure access to the site.

GitHub Pages Update Delay:

Initially, I faced an issue where changes made to the repository weren’t reflected immediately on the live site. After pushing updates, the changes would sometimes take a while to appear. This was resolved by waiting a bit longer and occasionally refreshing the GitHub Pages section in the repository settings to force the update.

Host the Site on GitHub Pages

Steps to Host:

  • After creating your HTML page, push it to a GitHub repository.
  • Go to the repository settings, scroll down to the GitHub Pages section, and select the branch you want to use (usually main or master).
  • GitHub will provide you with a URL (e.g., https://username.github.io/repository-name) where your site will be accessible.
